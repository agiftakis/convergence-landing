01 // The Problem

One AI is a single point of failure

When you ask one AI a question, you get one perspective. That model can be confidently wrong. It can miss edge cases, hallucinate facts, fixate on one approach, or pull from biased training data.

Convergence fixes this by orchestrating a structured debate between multiple frontier models from different companies. Weak answers get challenged. Strong ones get reinforced. The final synthesized output incorporates the best ideas from all of them — and is meaningfully better than any single model's first attempt.

Built as a CEN 3031 Software Engineering project, the entire orchestration logic lives in a Python engine, wrapped in a native Tauri v2 desktop shell. AI debating AI is the product.

convergence ~ debate.log
# Round 1 — independent answers $ claude → "Use approach A" $ gpt-4 → "Approach B is better" $ gemini → "Approach A, but..." $ grok → "Hybrid of A + B"
# Round 2 — critique & refine › models converge on hybrid › CONVERGENCE DETECTED
# Final synthesis › output: unified answer ✓
02 // How It Works

From task to consensus in minutes

01
You submit a task "Write Python code to convert .epub files to .pdf" — anything from code to research questions.
02
Each model answers independently Round 1: every configured AI writes its first answer in isolation, no knowledge of the others.
03
Models critique each other Round 2+: every AI sees the others' answers, critiques them, refines its own response.
04
The engine detects convergence When the models start agreeing on a core answer, the debate stops automatically (safety cap: 7 rounds).
05
Final synthesis A single, unified answer combining the strongest ideas from every model.
